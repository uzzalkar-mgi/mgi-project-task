# Real-time (Reverb) — activation guide

Scaffolding is in place; nothing is active yet (no packages installed, JS bootstrap untouched, so the current build is unaffected). Follow these steps **when you're ready** to turn on live updates.

## 1. Install server package
```bash
composer require laravel/reverb
php artisan reverb:install     # sets BROADCAST_CONNECTION=reverb + REVERB_* env + config
```

## 2. Install client packages
```bash
npm install --save laravel-echo pusher-js
```

## 3. Environment (.env)
```
BROADCAST_CONNECTION=reverb
REVERB_APP_ID=...            # generated by reverb:install
REVERB_APP_KEY=...
REVERB_APP_SECRET=...
REVERB_HOST="your-domain"
REVERB_PORT=8080
REVERB_SCHEME=https

# exposed to the browser (Vite)
VITE_REVERB_APP_KEY="${REVERB_APP_KEY}"
VITE_REVERB_HOST="${REVERB_HOST}"
VITE_REVERB_PORT="${REVERB_PORT}"
VITE_REVERB_SCHEME="${REVERB_SCHEME}"
```

## 4. Wire the client
```bash
mv resources/js/echo.js.example resources/js/echo.js
```
In `resources/js/app.jsx`, after the Inertia `createInertiaApp`, add:
```js
import { initEcho } from './echo';
initEcho();
```
(`initEcho` is a no-op if `VITE_REVERB_APP_KEY` is unset, so it's safe.)

## 5. Channel auth (`routes/channels.php`)
```php
Broadcast::channel('task.{uuid}', fn ($user, $uuid) => true);           // tighten as needed
Broadcast::channel('notifications.{id}', fn ($user, $id) => (int) $user->id === (int) $id);
```
Ensure `BroadcastServiceProvider` / `channels.php` is loaded (Laravel 11+: `->withBroadcasting(__DIR__.'/../routes/channels.php')` in bootstrap/app.php).

## 6. Dispatch events (backend)
`app/Events/RealtimeEvent.php` is ready. Example — after a task status change in `TaskController`:
```php
\App\Events\RealtimeEvent::dispatch('task.'.$task->uuid, 'TaskUpdated', ['uuid' => $task->uuid]);
```
With `BROADCAST_CONNECTION=null` these dispatches are safe no-ops, so you can add them before finishing setup.

## 7. Listen (frontend)
See the example in `echo.js` — subscribe on a page with `window.Echo.private('task.'+uuid).listen('.TaskUpdated', …)` and `router.reload({ only: [...] })`.

## 8. Run the socket server (persistent process)
```bash
php artisan reverb:start          # dev
# production: run under supervisor/systemd, e.g. `php artisan reverb:start --host=0.0.0.0 --port=8080`
```
Also proxy the WebSocket port through your web server (nginx `location /app { proxy_pass ... }`).

## Fallback
If the Reverb process is down or env is unset, the app works exactly as today — live updates just don't fire. Nothing breaks.
